# Редизайн журнала питания (май 2026)

## Интерактивный прототип
- Прототип реализован в рабочем UI страницы журнала: маршрут `/` (компонент `ChatPage`).
- Новая структура интерактивна: группировка записей по времени, контекстная помощь в композере, единый CTA, компактный Nightscout-блок.

## Изменения иерархии и визуальных приоритетов
- Основной порядок в строке записи: время -> название -> мета-строка -> макро/ккал блок.
- Название записи повышено в иерархии (крупнее и контрастнее мета-информации).
- Мета-строка переведена во вторичный слой: `источник / статус / масса|количество`.
- Записи с одинаковым временем сгруппированы в связанные мини-группы с общей визуальной осью.
- Макросы и калории сведены в компактный двухстрочный блок:
  - строка 1: `XX г угл · YYYY ккал`
  - строка 2: `B/F/C` и мини macro-bar.
- Подсказки из постоянного текста перенесены в контекстную помощь:
  - показываются при фокусе в поле ввода;
  - показываются в пустом дне;
  - скрыты по умолчанию.
- Блок Nightscout уплотнен и встроен в верхнюю область как компактная строка статуса.
- Нижняя панель ввода получила один главный CTA:
  - `Добавить запись` для текстового ввода;
  - `Оценить фото` при наличии прикрепленных фото.

## Тестовые сценарии usability
1. Контекстная помощь
- Открыть журнал с непустым днем: подсказки скрыты.
- Сфокусироваться в поле ввода: подсказки появляются.
- Снять фокус: подсказки скрываются.
- Открыть день без записей: подсказки видны даже без фокуса.

2. Группировка по времени
- Создать две записи с одинаковым временем.
- Проверить, что они отображаются как связанная мини-группа.
- Убедиться, что время явно показано для первой записи группы.

3. Быстрое считывание макросов
- Для нескольких записей проверить читаемость формата:
  - строка 1: углеводы + ккал;
  - строка 2: B/F/C + mini bar.
- Проверить, что пользователь может сравнить 3 соседние записи без открытия правой панели.

4. Ввод и CTA
- Ввести текст и нажать основной CTA: запись создается.
- Добавить фото, не вводя текст: CTA переключается в `Оценить фото`.
- Нажать CTA `Оценить фото`: запускается оценка фото.

5. Компактный Nightscout
- При настроенном NS проверить отображение несинхронизированных записей и времени последнего обновления.
- Нажать `Синхронизировать` и проверить обновление короткого результата отправки.

## Метрики успеха
- Время поиска информации (Time-to-Info):
  - измерение: медиана времени от появления списка до ответа "сколько углеводов и ккал в нужной записи".
  - целевой ориентир: снижение на 20%+ относительно предыдущей версии.
- Ошибки ввода (Input Error Rate):
  - измерение: доля неуспешных попыток (пустой submit, неверный сценарий CTA, отмена после ошибочного действия).
  - целевой ориентир: снижение на 15%+.
- Удовлетворенность пользователей (CSAT):
  - измерение: опрос после 3-5 минут работы (шкала 1-5) по понятности структуры, скорости ввода, читаемости макросов.
  - целевой ориентир: средний балл >= 4.2/5.
