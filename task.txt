Mathematical model v1:
Implement a conservative, robust, display-only model.

Definitions:
- raw CGM: Nightscout CGM value
- fingerstick: manual capillary glucose entry
- derived normalized glucose: display estimate from calibration model
- never overwrite raw

Fingerstick matching:
For each fingerstick reading:
- find CGM points in window `[fingerstick_time - 20min, fingerstick_time + 20min]`
- estimate local CGM value at fingerstick time by interpolation or nearest point
- compute local slope around fingerstick
- exclude from calibration if:
  - no nearby CGM
  - large gap
  - suspected compression/artifact
  - glucose is changing rapidly
  - duplicate fingerstick too close to another one

Initial thresholds:
- rapidly changing if absolute slope > 0.08 mmol/L/min
- nearby max distance 15 minutes preferred, 20 minutes fallback

Residual:
`residual = fingerstick_glucose - raw_cgm_at_time`

Calibration model:
Start simple and robust:
`normalized = raw + b0 + b1 * sensor_age_days`

Fit:
- robust linear regression residual ~ sensor_age_days
- if fewer than 3 valid fingersticks: use median offset only
- if fewer than 2 valid fingersticks: do not produce normalized values, only show raw and “недостаточно данных”
- use robust statistics; outliers should not dominate
- cap correction magnitude initially:
  - max absolute offset correction 3.0 mmol/L
  - max drift correction 0.5 mmol/L/day
  - if exceeded, mark low confidence instead of silently applying aggressive correction

Quality metrics:
For each sensor:
- fingerstick_count
- valid_calibration_points
- median_bias_mmol_l
- MAD/MARD if possible
- drift_mmol_l_per_day
- residual_std_or_mad
- missing_data_pct
- suspected_compression_count
- noise_score
- quality_score 0–100
- confidence: low/medium/high

Smoothing:
Implement only conservative smoothing for display:
- do not smooth raw data
- create smoothed display series
- use a robust rolling median or Hampel filter for isolated spikes
- then optional small moving average / EWMA
- preserve real meal/insulin trends
- never remove long sustained changes

Artifact detection:
Mark suspected compression lows:
- mostly nighttime or long rest periods if available
- sharp downward V-shape
- drop rate exceeds threshold
- recovery within 15–60 min
- no nearby insulin/food explanation
- not supported by fingerstick if available

Store flags:
- `compression_suspected`
- `jump_suspected`
- `gap`
- `low_confidence_calibration`
- `end_of_life_noise`

Sensor end-of-life:
Compute sensor age from `sensor_session.started_at`.
Show warning if:
- age >= 12 days
- noise_score increases
- residual variance increases
- drift magnitude increases
